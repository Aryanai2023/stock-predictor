What is Cloud Computing?

The on-demand delivery of IT resources over the internet with pay-as-you-go pricing.

Key Cloud Benefits

  • Agility: Innovate and deploy applications faster.
  • Elasticity: Scale resources (servers, etc.) up or down instantly based on demand.
  • CapEx vs. OpEx: Trade Capital Expense (buying hardware) for Variable Expense (paying for what you use).
  • Global Reach: Deploy applications in multiple geographic regions in minutes.

The 6 Pillars (Well-Architected Framework)

  1. Operational Excellence: Run and monitor systems to deliver business value.
  2. Security: Protect information, systems, and assets.
  3. Reliability: Ensure workloads perform their function correctly and consistently.
  4. Performance Efficiency: Use computing resources efficiently.
  5. Cost Optimization: Avoid or eliminate unneeded costs.
  6. Sustainability: Minimize the environmental impacts of running cloud workloads.

üîê Domain 2: Security & Compliance

The Shared Responsibility Model

  • AWS (Security OF the Cloud): Responsible for the physical hardware, global infrastructure, and the virtualization layer.
  • Customer (Security IN the Cloud): Responsible for their data, user access, OS, network/firewall configurations, and encryption.

Core Security: IAM (Identity and Access Management)

  • Users: An end-user (person or application). Best practice: Don't use the Root user for daily tasks.
  • Groups: A collection of users. Simplifies permission management.
  • Roles: Assumed by users or services to get temporary permissions.
  • Policies: JSON documents that explicitly define permissions (e.g., "Allow S3 Read Access").

Key Security Services

  • AWS Shield: Managed Distributed Denial of Service (DDoS) protection. (Standard is free).
  • AWS WAF (Web Application Firewall): Protects web apps from common exploits (e.g., SQL injection).
  • Amazon GuardDuty: Intelligent threat detection that monitors for malicious activity.
  • AWS Artifact: On-demand access to AWS's compliance reports (e.g., PCI, SOC, HIPAA).

üíª Domain 3: Technology & Services

AWS Global Infrastructure

  • Regions: Physically separate geographic areas in the world (e.g., N. Virginia).
  • Availability Zones (AZs): One or more discrete data centers within a Region. They are isolated from each other for fault tolerance.
  • Edge Locations: Cache content for CloudFront (CDN) to deliver data to users with low latency.

Core Compute Services

  • EC2 (Elastic Compute Cloud): Virtual servers (virtual machines) in the cloud.
  • Lambda: Serverless compute. Run code without managing servers.
  • Elastic Beanstalk: Platform as a Service (PaaS). You upload your code, and Beanstalk handles the deployment and scaling.

Core Storage Services

  • S3 (Simple Storage Service): Infinitely scalable object storage (for files, backups, website hosting).
  • EBS (Elastic Block Store): A block storage volume for a single EC2 instance (like a virtual hard drive).
  • EFS (Elastic File System): A file storage service that can be shared with multiple EC2 instances.

Core Database Services

  • RDS (Relational Database Service): A managed service for relational (SQL) databases (e.g., MySQL, PostgreSQL).
  • DynamoDB: A fully managed, high-performance NoSQL (key-value) database.

Core Networking & Monitoring

  • VPC (Virtual Private Cloud): Your own logically isolated section of the AWS cloud (your private virtual network).
  • Route 53: A highly available Domain Name System (DNS) web service.
  • CloudFront: A global Content Delivery Network (CDN) that speeds up content delivery.
  • CloudWatch: A monitoring service. Collects logs, metrics, and events. Can set alarms.
  • CloudTrail: Logs all API calls in your account. Answers "Who did what, and when?"

üßæ Domain 4: Billing & Support

Pricing Models

  • On-Demand: Pay by the hour/second. No commitment.
  • Reserved Instances (RIs): 1-3 year commitment for a large discount on services like EC2 and RDS.
  • Spot: Bid on spare EC2 capacity. Cheapest (up to 90% off), but can be terminated by AWS.

Cost Management Tools

  • AWS Budgets: Set alerts when your costs or usage exceed a threshold.
  • AWS Cost Explorer: Visualize and analyze your historical costs and usage.
  • AWS Pricing Calculator: Estimate the cost of your planned architecture before you build it.

Support Plans

  • Basic: Free. Billing and account support only.
  • Developer: For testing. Business-hour email access to tech support.
  • Business: For production. 24/7 phone, email, and chat access to tech support.
  • Enterprise: For mission-critical. Provides a dedicated Technical Account Manager (TAM).
